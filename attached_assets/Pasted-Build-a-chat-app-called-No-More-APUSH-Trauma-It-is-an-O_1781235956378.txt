IMPLEMENTATION. Must do:
- Read the OpenAI API key from environment variables. NEVER hardcode it.
  Python: `os.environ["OPENAI_API_KEY"]`
  Node/TypeScript: `process.env.OPENAI_API_KEY`
  Replit's Secrets are exposed as env vars, so this is how the app reads the key.
- Use the OpenAI Assistants API with the `file_search` tool.
- OpenAI SDK v2 API paths: `client.vector_stores` is TOP-LEVEL, not `client.beta.vector_stores`. Assistants and threads are still under `client.beta` (`client.beta.assistants`, `client.beta.threads`).
- Load `vector_store_id` from `config.json` and keep this vector store attached and unchanged — it already holds the full corpus, so do not rebuild or re-upload the corpus.
- Let users upload their own documents in the chat. Upload each file to OpenAI with `purpose="assistants"`, then attach it to that user's message using `attachments=[{"file_id": file_id, "tools": [{"type": "file_search"}]}]`. The Assistants API then searches BOTH the existing corpus vector store and the uploaded file, so the original corpus is always used and uploads stay scoped to that conversation. `file_search` reads text documents (PDF, txt, docx, md), not images.
- At startup, in a background thread: create a new assistant using the assistant name and system prompt above, then attach the existing `vector_store_id`. This lets the app start immediately for Replit's health check.
- Chat interface: streaming responses with citations.
